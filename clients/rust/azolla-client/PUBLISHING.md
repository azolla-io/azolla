This document outlines how to publish the azolla-client crate to crates.io.

- [x] **Licenses**: MIT and Apache 2.0 license files included
- [x] **Feature flags**: Proper feature flag structure for internal/external usage

## Key Features

### 1. Type-Safe Arguments
- Compile-time argument validation
- Automatic JSON serialization/deserialization
- Support for all Rust types that implement Serialize/Deserialize

### 2. Proc Macro Support
- `#[azolla_task]` attribute for zero-boilerplate task definitions
- Automatic task struct generation
- Type-safe argument extraction

### 3. Clean API
- Builder patterns for configuration
- Async/await ready
- Comprehensive error handling

### 4. Minimal Dependencies
- Only essential dependencies included
- Optional features for reducing binary size
- No server-side dependencies pulled in

## Dependencies Analysis

Core dependencies (all essential):
- `tokio`: Async runtime (minimal features)
- `tonic`: gRPC client
- `prost`: Protocol buffer support
- `serde`/`serde_json`: Serialization
- `thiserror`: Error handling
- `async-trait`: Trait support
- `log`: Logging interface
- `azolla-macros`: Proc macro support

Build dependencies:
- `tonic-build`: Protocol buffer compilation

Dev dependencies:
- `tokio-test`: Testing utilities

## Publishing Steps

1. **Version Check**: Ensure version is set correctly in Cargo.toml
2. **Final Tests**: Run all tests including doctests
   ```bash
   cargo test --all-features
   cargo test --doc
   ```
3. **Example Verification**: Ensure examples work
   ```bash
   cargo run --example basic_usage
   ```
4. **Package Creation**: Create the package locally
   ```bash
   cargo package
   ```
5. **Package Inspection**: Review the generated package
   ```bash
   cargo package --list
   ```
6. **Publish**: Upload to crates.io
   ```bash
   cargo publish
   ```

## Usage for End Users

Users can add this to their Cargo.toml:

```toml
[dependencies]
azolla-client = "0.1.0"
tokio = { version = "1.0", features = ["rt-multi-thread", "macros"] }
serde_json = "1.0"
```

Basic usage:

```rust
use azolla_client::{azolla_task, Client, Worker, TaskError};
use serde_json::{json, Value};

#[azolla_task]
async fn my_task(name: String, value: i32) -> Result<Value, TaskError> {
    Ok(json!({
        "result": format!("{}: {}", name, value)
    }))
}

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    // Client usage
    let client = Client::builder()
        .endpoint("http://localhost:52710")
        .domain("my-domain")
        .build()
        .await?;
        
    let task = client
        .submit_task("my_task")
        .args(("test".to_string(), 42))?
        .submit()
        .await?;
        
    // Worker usage
    let worker = Worker::builder()
        .orchestrator("localhost:52710")
        .domain("my-domain")
        .register_task(MyTaskTask) // Generated by proc macro
        .build()
        .await?;
        
    worker.run().await?;
    
    Ok(())
}
```
